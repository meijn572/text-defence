## 五、核心发现

1. **BERT 对中文对抗扰动具有意外高鲁棒性**：所有攻击 F1 ≥ 0.95
2. **四通道融合无显著提升**：所有差异 < 0.01，统计不显著
3. **正规化降低性能**：baseline_aug 全面低于 baseline
4. **字符乱序 F1 最高**：BERT 子词分词天然容错
5. **GPU 全程不可用**：subprocess CUDA 冲突 + pd.read_csv/torch 顺序问题
6. **必须以 CPU 运行**：CSV 必须在所有 ML 库导入前读取

## 六、可复现运行流程

### 前提
```
当前数据: spam_data.csv = 2,000条精简数据
环境: Python 3.x + CPU only
目录: d:\3_second\big_data\work\text-defense
```

### 命令

```powershell
# 实验01：对抗样本生成（~30秒）
Push-Location "d:\3_second\big_data\work\text-defense"
python -u run_small_exp.py     # Ctrl+C 跳过后续步骤

# 实验02：基线BERT训练（~13分钟）
& { Set-Location d:\3_second\big_data\work\text-defense; python -u train_baseline_direct.py }

# 实验03+04：融合训练+评测（~12分钟）
& { Set-Location d:\3_second\big_data\work\text-defense; python -u run_fusion_eval.py }

# 强攻击加测（~3分钟）
& { Set-Location d:\3_second\big_data\work\text-defense; python -u run_strong_attack.py }
```

### 工作脚本说明

| 脚本 | 功能 | 关键 |
|------|------|------|
| `run_small_exp.py` | 01 对抗样本生成 | subprocess可用 |
| `train_baseline_direct.py` | 02 BERT训练 | CSV先于torch |
| `run_fusion_eval.py` | 03+04 融合+评测 | CSV先于torch |
| `run_strong_attack.py` | 强攻击加测 | 生成J/K/L+对比 |

> ⚠️ **铁律**：CSV 在 torch 前读；不通过 subprocess 调用含 torch 脚本
