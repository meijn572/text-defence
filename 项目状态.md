## 五、核心发现

1. **BERT 对中文对抗扰动具有意外高鲁棒性**：所有攻击 F1 ≥ 0.95
2. **四通道融合无显著提升**：所有差异 < 0.01，统计不显著
3. **正规化降低性能**：baseline_aug 全面低于 baseline
4. **字符乱序 F1 最高**：BERT 子词分词天然容错
5. **传统基准仍有竞争力**：Word2Vec-c+GBDT 在当前测试集上 `ALL F1 = 0.9753`
6. **GPU 已在整合脚本中可用**：`run_all.py` 不再依赖 subprocess，可使用 CUDA 版 PyTorch
7. **CPU 可复现传统基准**：`classic_baselines.py` 可快速复现五个经典基准
8. **Gradio 前端可展示系统能力**：`app.py` 支持 CPU/GPU 自适应，提供短信检测、微信式聊天演示、对抗攻击演示和实验结果查看
9. **内容类型分析已补充**：`content_type_analysis.py` 使用关键词弱监督划分垃圾短信内容类型，用于分析贷款、中奖、营销、钓鱼链接等类型分布
10. **内容类型模型提升评估已接入**：`content_type_model_eval.py` 可生成 `content_type_model_recall.csv`，按内容类型比较 BERT 与 Fusion 的垃圾召回率

输出文件：

- `results/content_type_distribution.csv`

- `results/content_type_model_recall.csv`：按内容类型统计 `BERT_recall`、`Fusion_recall`、漏检数和 `Fusion_minus_BERT`

CPU 环境下默认每个内容类型最多抽样 30 条进行快速评估；GPU 环境下可进行全量评估。

### 算法框架与输入输出

```text
输入：中文短信文本 x
        │
        ├─ 攻击模块：A-I 基础攻击 + J/K/L 强攻击
        │       输出：对抗短信 x_adv
        │
        ├─ 防御预处理：Unicode 清洗、零宽字符处理、繁简/文本正规化
        │       输出：规范化文本 x_norm
        │
        ├─ 四通道特征提取
        │   ├─ BERT 文本通道      → 768 维语义特征
        │   ├─ 拼音 CNN 通道      → 256 维语音特征
        │   ├─ 视觉 ResNet 通道   → 512 维字形特征
        │   └─ BoW 字袋通道       → 128 维字符集合特征
        │
        ├─ 融合分类器：Concat(1664) → MLP
        │
        └─ 输出：正常/垃圾概率 + 预测标签
```

| 模块 | 输入 | 输出 |
|------|------|------|
| 攻击模块 | 原始短信 | 对抗短信 |
| 正规化模块 | 原始/对抗短信 | 规范化文本 |
| 四通道模型 | 规范化文本 | 1664 维融合特征 |
| 分类器 | 融合特征 | 正常/垃圾概率 |
| 评测模块 | 预测结果与真实标签 | Accuracy、Precision、Recall、F1 |
| 前端展示 | 用户输入短信 | 检测结果、攻击结果、实验表格 |
