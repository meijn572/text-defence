# 实验结论记录 v2.0

> 文本检测对抗攻防系统 | 2026-06-15 | GPU 训练 | 消融实验完成

---

## 一、实验设置

| 项目 | 说明 |
|------|------|
| 数据集 | 中文垃圾短信 2,000 条（1:1 均衡） |
| 训练集 | 1,400 条 → A/B 增强 → 2,800 条 |
| 测试集 | 600 条原始 + 300×9 对抗 + 300×3 强攻击 = 3,900 条 |
| 训练设备 | **GPU (CUDA)**，相比 CPU 加速显著 |
| 训练轮数 | BERT 1 epoch，融合模型 2 epochs，BERT+正规化 1 epoch |
| 消融方法 | Zero-out ablation（通道置零） |

## 二、模型训练结果

| 模型 | Val Acc | Val F1 | 耗时 |
|------|:-------:|:------:|:----:|
| 朴素 BERT | 95.71% | 0.9710 | 47s |
| BERT + 正规化 | 95.71% | 0.9710 | 47s |
| 四通道融合 | 95.36% | 0.9691 | 228s |

> 融合模型训练时间 ≈ BERT 基线的 4.8 倍，但验证集性能相当

### 3.2 消融实验分析（四通道融合模型）

#### 文本通道置零（-text）

| 子集 | F1 | Acc | 下降幅度 |
|-----|:--:|:---:|:-------:|
| 原始样本 | 0.3168 | 0.5183 | -65.4% |
| 对抗_A | 0.3099 | 0.1833 | -68.3% |
| 对抗_B | 0.4334 | 0.2767 | -55.5% |
| 对抗_C | 0.3425 | 0.2067 | -63.9% |
| 对抗_I | 0.3516 | 0.2133 | -63.3% |

**结论**：文本通道置零后 F1 普遍下降 **55-68%**，说明 BERT 是模型核心。

#### 拼音通道置零（-phonetic）

| 子集 | F1 | Acc | 变化 |
|-----|:--:|:---:|:----:|
| 原始样本 | 0.9052 | 0.9033 | -0.1% |
| 对抗_A | 0.9779 | 0.9567 | 0.0% |
| 对抗_F (音近字) | 0.9796 | 0.9600 | ±0.0% |
| 对抗_I | 0.9779 | 0.9567 | -0.1% |

**结论**：拼音通道对性能无显著影响（Δ < 0.15%），包括预期强相关的音近字攻击。

#### 视觉通道置零（-visual）

| 子集 | F1 | Acc | 变化 |
|-----|:--:|:---:|:----:|
| 原始样本 | 0.9040 | 0.9033 | -0.0% |
| 对抗_A | 0.9779 | 0.9567 | 0.0% |
| 对抗_G (形近字) | 0.9565 | 0.9167 | -1.0% |
| 对抗_I | 0.9761 | 0.9533 | -0.2% |

**结论**：视觉通道影响最小（Δ < 1%），形近字攻击上也无显著贡献。

#### 字袋通道置零（-bow）

| 子集 | F1 | Acc | 变化 |
|-----|:--:|:---:|:----:|
| 原始样本 | 0.9076 | 0.9050 | +0.3% |
| 对抗_A | 0.9779 | 0.9567 | 0.0% |
| 对抗_I | 0.9779 | 0.9567 | ±0.0% |

**结论**：字袋通道置零后性能反而略升（可能减少过拟合噪声），与设计预期不符。

### 3.3 经典基准算法复现实验（CPU 友好版）

为补充与传统垃圾文本检测方法的横向比较，项目新增 `classic_baselines.py`，在相同 `data/adversarial/train.csv` 与 `data/adversarial/test_full.csv` 上复现了五类经典基准算法。

| 基准算法 | 实现说明 | ALL F1 | Original F1 |
|---------|----------|:------:|:-----------:|
| GAS-lite | 字符共现图 + 风险传播 + LR，GAS 思想轻量近似 | 0.8866 | 0.7477 |
| Word2Vec-w+LR | jieba 分词 + 词级 Word2Vec + IDF 池化 + LR | 0.9642 | 0.8929 |
| Word2Vec-c+LR | 字符级 Word2Vec + IDF 池化 + LR | 0.9003 | 0.8299 |
| Word2Vec-c+GBDT | 字符级 Word2Vec + IDF 池化 + GBDT | **0.9753** | 0.8718 |
| Doc2Vec-c+GBDT | 字符级 Doc2Vec + GBDT | 0.8525 | 0.6783 |

完整结果文件：

- `results/classic_baseline_results.csv`
- `results/classic_baseline_f1_pivot.csv`

**复现说明**：

- `GAS-lite` 是 CPU 友好的图结构近似基线，保留“利用历史垃圾文本关系图”的思想，但没有实现完整 GCN 端到端训练，因此不能等同于完整 GAS 原方法。
- `Word2Vec` / `Doc2Vec` 系列为便于 CPU 快速复现，使用 IDF 加权池化生成句子向量，并非完整自注意力池化版本。
- 该脚本依赖 `gensim`、`jieba`、`scikit-learn`、`scipy`、`numpy`、`pandas`，已补充至依赖清单。

**结果观察**：

1. `Word2Vec-c+GBDT` 是当前传统基准中最强模型，`ALL F1 = 0.9753`，说明字符级静态向量结合非线性树模型在当前数据分布下非常有竞争力。
2. `Word2Vec-w+LR` 在原始样本上达到 `F1 = 0.8929`，接近 BERT / 四通道融合模型，说明普通垃圾短信检测中词级模板特征仍然很强。
3. `GAS-lite` 与 `Word2Vec-c+LR` 接近，但明显弱于 `Word2Vec-c+GBDT`，说明仅依赖历史字符共现关系不足以稳定处理复杂文本变体。
4. `Doc2Vec-c+GBDT` 表现最弱，说明当前短短信、小数据场景下句向量学习不够稳定。
5. 当前 `test_full.csv` 标签分布明显不均衡（正常 300 / 垃圾 3000），攻击子集主要由垃圾样本构成，因此攻击集上的高 F1 不能直接等同于真实开放场景下的全面鲁棒性，后续仍需补充平衡测试集与 ASR 攻击成功率。

## 四、核心结论

### 结论 1：**文本通道（BERT）是融合模型的唯一核心驱动力**

- 置零后 F1 下跌 **55-68%**，模型基本失效
- 其他三个通道置零，性能几乎无变化（Δ < 1.5%）
- **消融学习**：多通道融合的收益主要来自 BERT 的语义理解能力

### 结论 2：**朴素 BERT 对中文对抗扰动具有意外的高鲁棒性**

- 平均 F1 = **0.9565**，所有 13 个子集 F1 ≥ 0.90
- 即使在强攻击（J/K/L）下仍保持 **0.96+ 的 F1**
- **原因分析**：
  - BERT 的 WordPiece 分词对字符级干扰天然鲁棒
  - 注意力机制能从上下文推断原义，不依赖单字
  - 中文垃圾短信任务本身特征明显，不需特殊防御

### 结论 3：**正规化预处理反而略降性能**

- BERT + 正规化平均 F1 = 0.9676（实际与朴素 BERT 相当）
- 原始样本上明显下降：0.8644 vs 0.9016（↓ 4.1%）
- **原因**：正规化在清理对抗扰动的同时，也可能损伤有用文本特征

### 结论 4：**四通道融合有小幅提升，但不统计显著**

- 平均 F1 = 0.9708，相比朴素 BERT 的 0.9565 提升 **1.5%**
- 在 8/13 子集上超过朴素 BERT，在 3/13 子集下降
- **为何提升有限**：
  - 拼音、视觉、字袋通道单独贡献极小
  - BERT 已经处理得很好，多通道难以进一步改进
  - 2k 数据量可能导致多通道模型欠拟合

### 结论 5：**传统基准仍具有很强竞争力**

- `Word2Vec-c+GBDT` 在当前 `test_full.csv` 上达到 `ALL F1 = 0.9753`，高于多数轻量基准，也与 BERT/Fusion 形成强竞争。
- `Word2Vec-w+LR` 原始样本 `F1 = 0.8929`，接近 BERT/Fusion，说明短信任务中关键词、URL、电话、营销模板等浅层特征非常强。
- 因此不能简单声称“四通道模型显著优于传统方法”。更准确的结论是：BERT/Fusion 的优势主要体现在上下文语义建模、多通道中文扰动建模和攻防一体化框架，而不是在当前所有 F1 指标上压倒传统模型。
- 当前基准复现实验为 CPU 友好版，且测试集类别不平衡，后续应补充强攻击 J/K/L、ASR、平衡测试集和多随机种子实验。

### 结论 6：**强攻击（J/K/L）难度有限**

- J 强乱序：F1 = 0.9831（相比普通乱序无显著差异）
- K 强音近：F1 = 0.9865（甚至优于普通音近）
- L 混合攻击：F1 = 0.9813（介于单个攻击之间）

**说明**：当前强度的攻击对 BERT 没有构成真正威胁

## 五、消融实验的方法论反思

### 使用的方法：Zero-out Ablation（通道置零）

**优点**：
- 实现简单，快速得出各通道相对贡献
- 能定性反映不同通道的重要性

**缺点**：
- 导致分布偏移：置零后特征维度不匹配，导致融合层输入异常
- 可能高估/低估通道贡献：例如字袋通道置零时 F1 反而升高
- 不严谨：更正确的做法是为三通道、二通道模型单独训练，再对比

### 改进方向

1. **正规化训练**：分别训练 3/2/1 通道模型（不置零），对比性能差异
2. **通道复制**：用复制通道替代置零，保持输入维度一致
3. **可视化分析**：观察融合层权重，定量计算各通道注意力分数

## 七、讨论与研究价值

### 与架构设计的对比

| 设计预期 | 实验结果 | 学术启示 |
|---------|---------|---------|
| 音近字攻击 F1 ↓ 至 55% | 实际 96.4% | BERT 从上下文推断，不依赖单字音韵 |
| 视觉通道补偿形近字 | 形近字仍主要靠 BERT | 形近扰动属于表层，BERT 语义理解天然免疫 |
| 多通道协同增强鲁棒性 | 各通道贡献 < 1.5% | 预训练语言模型已高度鲁棒，多模态补益有限 |

### 值得深入的研究问题

1. **是否存在 BERT 的致命弱点？**
   - 尝试语义反向攻击（e.g. "这是垃圾" ↔ "这不是垃圾"）
   - 组织攻击和语义攻击的组合

2. **为何拼音通道无效？**
   - 拼音 CNN 设计是否有缺陷？
   - 音近字替换是否不足以破坏拼音序列信息？

3. **多通道融合的正确打开方式是什么？**
   - 是否需要更复杂的特征融合策略（e.g. cross-attention)?
   - 数据量和模型容量的平衡点在哪？

## 八、诚实总结

> 在当前实验条件下（2k 数据 + GPU 训练 + 消融实验），
> **预训练 BERT 就是最优选择**。四通道融合架构在理论上合理，
> 但在实践中多通道贡献均不显著。
>
> **这本身是重要的科学发现**：
> - ✅ 预训练语言模型对中文对抗扰动具有意外的高鲁棒性
> - ✅ 多模态防御在当前任务上不是瓶颈
> - ✅ 表层扰动（乱序、同形字）对语义理解影响有限
>
> **后续可探索方向**：
> - 更强的语义级对抗攻击
> - 更大规模数据的融合模型效果
> - 跨语言/跨域的鲁棒性评估
